# ALMA v3 — Modular Autonomous Language Agent

**Core + Extensions Architecture**

A minimal, elegant core that proves the hypothesis — with well-defined extension points that add complexity *only when they demonstrably improve results*.

---

## Design Philosophy

> **Simple core, optional complexity, empirical justification.**

The architecture has three layers:

| Layer | Purpose | Complexity | When to Add |
|-------|---------|------------|-------------|
| **Core** | Prove the hypothesis | Minimal (800 LOC) | Always present |
| **Extensions** | Improve performance | Moderate (500-1000 LOC each) | When benchmarks show need |
| **Applications** | Real-world use cases | Variable | After core + extensions validated |

**Key principle:** Every extension is:
- **Modular** (can be added/removed without breaking core)
- **Measurable** (has clear benchmarks for success)
- **Optional** (core works without it)

### Component Specifications

#### Planner (Core)
```python
class Planner(nn.Module):
    """Minimal policy network."""
    
    def __init__(self, mem_dim=512, hidden_dim=256, latent_dim=256):
        super().__init__()
        # Simple 2-layer transformer
        encoder_layer = nn.TransformerEncoderLayer(
            d_model=mem_dim, nhead=8, dim_feedforward=1024,
            batch_first=True, norm_first=True
        )
        self.transformer = nn.TransformerEncoder(encoder_layer, num_layers=2)
        self.mode_emb = nn.Embedding(2, mem_dim)
        
        # Three output heads
        self.latent_head = nn.Linear(mem_dim, latent_dim)
        self.alpha_head = nn.Linear(mem_dim, 1)
        self.mode_head = nn.Linear(mem_dim, 2)
        
    def forward(self, memory, mode):
        mode_vec = self.mode_emb(mode).unsqueeze(1)
        x = torch.cat([memory, mode_vec], dim=1)
        out = self.transformer(x)
        pooled = out[:, -1, :]
        
        latent_mean = self.latent_head(pooled)
        alpha = torch.sigmoid(self.alpha_head(pooled)) * 2.0  # Range [0, 2]
        mode_logits = self.mode_head(pooled)
        
        return latent_mean, alpha, mode_logits
```

**Design choices:**
- 2 layers (not 4) — sufficient for learning, faster iteration
- Alpha as learned output (not frozen parameter)
- Mode for flat vs. hierarchical (can be disabled via config)

#### Bridge (Core)
```python
class Bridge(nn.Module):
    """Projection between planner and model spaces."""
    
    def __init__(self, latent_dim=256, model_dim=960, injection_layer=24):
        super().__init__()
        self.up_projection = nn.Linear(latent_dim, model_dim, bias=False)
        self.down_projection = nn.Linear(model_dim, latent_dim, bias=False)
        self.injection_layer = injection_layer
        
    def get_steering_vector(self, latent_goal, alpha):
        """Project latent goal to activation space, scale by alpha."""
        raw = self.up_projection(latent_goal)
        normalized = raw / (raw.norm(dim=-1, keepdim=True) + 1e-8)
        return normalized * alpha.unsqueeze(-1)
        
    def get_outcome_embedding(self, hidden_state):
        """Project model output back to planner space."""
        return self.down_projection(hidden_state)
```

**Design choices:**
- Single projection (no LoRA generation)
- Single injection layer (configurable)
- Alpha applied multiplicatively

#### Memory (Core)
```python
class VectorMemory:
    """Simple sliding window of outcome embeddings."""
    
    def __init__(self, size=32, dim=256):
        self.size = size
        self.buffer = deque(maxlen=size)
        self.dim = dim
        
    def append(self, outcome_embedding):
        self.buffer.append(outcome_embedding.detach().cpu())
        
    def novelty(self, query_embedding):
        """Compute cosine novelty against memory mean."""
        if len(self.buffer) < 2:
            return torch.tensor([1.0])
        memory_tensor = torch.stack(list(self.buffer))
        mean_embedding = memory_tensor.mean(dim=0)
        similarity = F.cosine_similarity(
            query_embedding.cpu().unsqueeze(0),
            mean_embedding.unsqueeze(0),
            dim=-1
        )
        return 1.0 - similarity
        
    def get_tensor(self):
        """Return memory as tensor for planner input."""
        if len(self.buffer) == 0:
            return torch.zeros(1, 1, self.dim)
        return torch.stack(list(self.buffer)).unsqueeze(0)
```

**Design choices:**
- Fixed-size deque (no LRU logic)
- CPU storage (saves GPU memory)
- Mean-based novelty (not k-NN, simpler)

#### Reward (Core)
```python
class RewardFunction:
    """Three-component reward: novelty, coherence, task relevance."""
    
    def __init__(self, config):
        self.w_novelty = config.get('novelty_weight', 1.0)
        self.w_coherence = config.get('coherence_weight', 0.5)
        self.w_relevance = config.get('relevance_weight', 0.0)
        self.target_embedding = None
        
        if self.w_relevance > 0 and config.get('goal_text'):
            self.target_embedding = self._encode_goal(config['goal_text'])
    
    def compute(self, outcome, coherence_loss, novelty):
        """Compute total reward and components."""
        reward = self.w_novelty * novelty - self.w_coherence * coherence_loss
        
        if self.w_relevance > 0 and self.target_embedding is not None:
            relevance = F.cosine_similarity(
                outcome, self.target_embedding, dim=-1
            ).item()
            reward += self.w_relevance * relevance
            
        return reward, {'novelty': novelty, 'coherence': -coherence_loss}
```

**Design choices:**
- Linear combination (no learned weights in core)
- Optional relevance (zero if no goal text)
- Returns decomposition for logging

#### Training Loop (Core)
```python
def train_step(agent, env, optimizer, config):
    """Single PPO update step."""
    # Collect batch of trajectories
    trajectories = []
    for _ in range(config['batch_size']):
        traj = rollout(agent, env, config['rollout_length'])
        trajectories.append(traj)
    
    # Compute returns and advantages (GAE)
    returns, advantages = compute_gae(trajectories, config['gamma'], config['lambda'])
    
    # PPO update
    for _ in range(config['ppo_epochs']):
        policy_loss, value_loss = ppo_loss(
            agent, trajectories, returns, advantages, config['clip_epsilon']
        )
        loss = policy_loss + 0.5 * value_loss
        
        optimizer.zero_grad()
        loss.backward()
        nn.utils.clip_grad_norm_(agent.parameters(), config['grad_clip'])
        optimizer.step()
    
    return loss.item()
```

**Design choices:**
- PPO (not REINFORCE) — stable, well-understood
- GAE for advantage estimation
- Configurable epochs, clip epsilon, etc.

### Extension 1: Multi-Layer Steering

**Problem:** Single-layer injection may not capture complex behaviors.

**Solution:** Inject at multiple layers with learned per-layer alphas.

```python
class MultiLayerBridge(Bridge):
    """Extension: Multi-layer activation addition."""
    
    def __init__(self, latent_dim=256, model_dim=960, injection_layers=[8, 16, 24]):
        super().__init__(latent_dim, model_dim)
        self.injection_layers = injection_layers
        # Per-layer projections
        self.projections = nn.ModuleList([
            nn.Linear(latent_dim, model_dim, bias=False)
            for _ in range(len(injection_layers))
        ])
        # Per-layer alphas (learned or planner-output)
        self.alpha_source = config.get('alpha_source', 'parameter')
        if self.alpha_source == 'parameter':
            self.layer_alphas = nn.Parameter(torch.ones(len(injection_layers)) * 0.3)
        # If 'planner', planner outputs vector of alphas
        
    def get_steering_vectors(self, latent_goal, alphas=None):
        """Return dict of {layer_idx: steering_vector}."""
        if alphas is None:
            alphas = self.layer_alphas
            
        vectors = {}
        for i, layer_idx in enumerate(self.injection_layers):
            raw = self.projections[i](latent_goal)
            normalized = raw / (raw.norm(dim=-1, keepdim=True) + 1e-8)
            vectors[layer_idx] = normalized * alphas[i]
        return vectors
```

**When to add:**
- Core plateau at reward < target after 1000 steps
- Single-layer steering shows mode collapse

**Success criteria:**
- +20% reward improvement
- Coherence maintained (perplexity < 3.0)

**Benchmark:** Run identical task with single vs. multi-layer, compare reward curves.

### Extension 2: Model Cascade

**Problem:** Single model is either too slow (1.7B) or too weak (135M).

**Solution:** Route tasks between models based on predicted complexity.

```python
class ModelCascade:
    """Extension: Dynamic model selection."""
    
    def __init__(self, models, router_config):
        self.models = nn.ModuleList(models)  # [SmolLM2-135M, SmolLM2-360M]
        self.costs = [0.0, 0.0]  # Can add API models with real costs
        self.latencies = [10, 50]  # ms per token
        
        # Router is a small MLP
        self.router = nn.Sequential(
            nn.Linear(task_dim, 64),
            nn.ReLU(),
            nn.Linear(64, len(models))
        )
        
    def select_model(self, task_embedding, budget=float('inf'), max_latency=float('inf')):
        """Select model based on task and constraints."""
        logits = self.router(task_embedding)
        
        # Mask models that violate constraints
        for i, (cost, latency) in enumerate(zip(self.costs, self.latencies)):
            if cost > budget or latency > max_latency:
                logits[:, i] = -float('inf')
                
        model_idx = Categorical(logits=logits).sample()
        return self.models[model_idx], model_idx
```

**When to add:**
- Core works but latency > 100ms/step
- Task success rate varies widely by complexity

**Success criteria:**
- −50% average latency
- Same or better task success rate

**Benchmark:** Mixed-complexity task suite, compare single-model vs. cascade.

### Extension 3: Human Feedback Channel

**Problem:** Purely intrinsic reward may not align with human preferences.

**Solution:** Add human feedback as an additional reward channel.

```python
class HumanFeedbackReward:
    """Extension: RLHF-style feedback integration."""
    
    def __init__(self, buffer_size=100):
        self.buffer = deque(maxlen=buffer_size)
        self.encoder = SentenceTransformer('all-MiniLM-L6-v2')
        
    def add_feedback(self, text, rating):
        """Store human feedback (rating: -1 to +1)."""
        embedding = self.encoder.encode(text, convert_to_tensor=True)
        self.buffer.append({'embedding': embedding, 'rating': rating})
        
    def get_reward(self, outcome_embedding):
        """Compute reward based on similarity to liked outcomes."""
        if len(self.buffer) < 5:
            return 0.0  # Not enough data
            
        liked = [b for b in self.buffer if b['rating'] > 0.5]
        disliked = [b for b in self.buffer if b['rating'] < -0.5]
        
        if not liked or not disliked:
            return 0.0
            
        liked_mean = torch.stack([b['embedding'] for b in liked]).mean(dim=0)
        disliked_mean = torch.stack([b['embedding'] for b in disliked]).mean(dim=0)
        
        sim_to_liked = F.cosine_similarity(outcome_embedding, liked_mean.unsqueeze(0))
        sim_to_disliked = F.cosine_similarity(outcome_embedding, disliked_mean.unsqueeze(0))
        
        return (sim_to_liked - sim_to_disliked).item()
```

**When to add:**
- Core produces coherent output but humans don't like it
- Style/quality preferences not captured by intrinsic reward

**Success criteria:**
- +15% human preference (A/B test)
- Feedback efficiency: < 20 samples to see improvement

**Benchmark:** Collect 100 human ratings, compare pre/post feedback integration.

### Extension 4: Skill Library

**Problem:** Trained planners are not reusable across sessions.

**Solution:** Standardized skill pack format with metadata and validation.

```python
class SkillLibrary:
    """Extension: Save, load, and manage trained skills."""
    
    def __init__(self, skills_dir="skills/"):
        self.skills_dir = Path(skills_dir)
        self.skills_dir.mkdir(exist_ok=True)
        
    def save_skill(self, planner, bridge, config, metrics, tags):
        """Export a trained skill."""
        skill = {
            'planner_state': planner.state_dict(),
            'bridge_state': bridge.state_dict(),
            'config': config,
            'metrics': metrics,  # {avg_reward, coherence, novelty, ...}
            'tags': tags,  # ['style', 'technical', 'biology']
            'created_at': datetime.now().isoformat()
        }
        skill_id = f"skill_{tags[0]}_{datetime.now().strftime('%Y%m%d_%H%M%S')}"
        torch.save(skill, self.skills_dir / f"{skill_id}.pt")
        return skill_id
        
    def load_skill(self, skill_id):
        """Load a skill for use."""
        skill = torch.load(self.skills_dir / f"{skill_id}.pt")
        return skill
        
    def search_skills(self, query_tags):
        """Find skills matching tags."""
        matches = []
        for skill_file in self.skills_dir.glob("*.pt"):
            skill = torch.load(skill_file)
            if any(tag in skill['tags'] for tag in query_tags):
                matches.append({
                    'id': skill_file.stem,
                    'tags': skill['tags'],
                    'metrics': skill['metrics'],
                    'created_at': skill['created_at']
                })
        return matches
```

**When to add:**
- Core produces transferable skills
- Need to manage multiple trained planners

**Success criteria:**
- Skills load and work without retraining
- Search finds relevant skills > 80% of time

**Benchmark:** Train 10 skills, reload each, measure performance degradation.

## Growth Paths — How the Design Evolves

The architecture supports multiple growth directions, each empirically justified:

### Path A: Performance Optimization

```
Core → Multi-Layer Steering → Model Cascade → Distillation
```

**Trigger:** Each step shows +performance on benchmark suite.  
**End state:** High-performance agent with optimal latency/quality tradeoff.

---

### Path B: Human Alignment

```
Core → Human Feedback → Preference Learning → Multi-Objective Reward
```

**Trigger:** Human preference scores below target.  
**End state:** Agent aligned with human values and preferences.

---

### Path C: Knowledge & Memory

```
Core → Vector Memory → Episodic Memory → Graph Memory
```

**Trigger:** Tasks requiring long-term reasoning or structured knowledge.  
**End state:** Agent with rich, queryable knowledge representation.

---

### Path D: Skill Ecosystem

```
Core → Skill Library → Skill Sharing → Community Repository
```

**Trigger:** Users creating and reusing skills.  
**End state:** Ecosystem of composable, shareable skill packs.

## Validation Protocol

Every addition (core or extension) follows this protocol:

### Phase 1: Implementation
- Build the component
- Write unit tests
- Document interface

### Phase 2: Benchmarking
- Run standard benchmark suite (see below)
- Compare to baseline (without the component)
- Measure: reward, coherence, latency, memory

### Phase 3: Decision
- **If improvement > threshold:** Merge into main branch
- **If improvement < threshold:** Keep as optional extension
- **If regression:** Remove or redesign

### Standard Benchmark Suite

| Benchmark | What it measures | Target |
|-----------|------------------|--------|
| **Style Transfer** | Can it learn and apply a style? | Classifier accuracy > 80% |
| **Concept Exploration** | Does it explore coherently toward a goal? | Goal relevance > 0.6 |
| **Multi-Step Coherence** | Can it maintain coherence over 10+ steps? | Avg perplexity < 3.0 |
| **Skill Transfer** | Do exported skills work on new inputs? | Performance drop < 10% |
| **Latency** | How fast is each step? | < 100ms on RTX 3060 |
