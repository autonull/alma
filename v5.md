## What "Autonomous Goal-Driven" Actually Means

### Autonomous ≠ Reactive

| Reactive (Current LLMs) | Autonomous (ALMA) |
|-------------------------|-------------------|
| Waits for user input | Generates its own objectives |
| Single-turn response | Multi-turn pursuit |
| No memory across turns | Accumulates knowledge |
| No self-direction | Revises own strategy |

### Goal-Driven ≠ Prompt-Following

| Prompt-Following | Goal-Driven |
|------------------|-------------|
| "Write about photosynthesis" | "Understand how plants convert light to energy" |
| One-shot output | Multi-step investigation |
| No success criterion | Knows when it's done |
| No revision | Self-corrects based on progress |

## The Actual Problem

**Current LLMs are fundamentally reactive.** They:
- Have no objectives beyond "complete the pattern"
- Cannot pursue a goal across multiple interactions
- Don't know what they don't know
- Can't revise their approach when stuck

**ALMA should solve this:** A system that:
1. **Generates its own goals** (not just follows prompts)
2. **Pursues goals over time** (multi-step, multi-turn)
3. **Tracks progress** (memory of what it's tried)
4. **Revises strategy** (learns from failure)
5. **Produces useful artifacts** (knowledge, answers, creations)

## The Goal Generator — Where Autonomy Lives

This is the heart of the system. It's what makes ALMA **autonomous** instead of **reactive**.

### Architecture

```python
class GoalGenerator(nn.Module):
    def __init__(self, config):
        # Encoder: reads current knowledge state
        self.knowledge_encoder = TransformerEncoder(...)
        
        # Goal heads: different types of goals
        self.explore_head = nn.Linear(hidden_dim, num_topics)
        self.explain_head = nn.Linear(hidden_dim, num_mechanisms)
        self.create_head = nn.Linear(hidden_dim, num_artifact_types)
        self.resolve_head = nn.Linear(hidden_dim, num_question_types)
        
        # Priority scorer: which goal to pursue now
        self.priority_head = nn.Linear(hidden_dim, 1)
        
    def forward(self, knowledge_state, memory):
        # Encode current state
        encoded = self.knowledge_encoder(knowledge_state)
        
        # Generate goal candidates
        candidates = {
            'explore': self.explore_head(encoded),
            'explain': self.explain_head(encoded),
            'create': self.create_head(encoded),
            'resolve': self.resolve_head(encoded),
        }
        
        # Score each candidate by priority
        priorities = self.priority_head(encoded)
        
        # Select top goal
        best_goal = self._select_goal(candidates, priorities, memory)
        
        return best_goal
```

### Goal Types

| Type | Description | Example |
|------|-------------|---------|
| **Explore** | Find new information about a topic | "What do I not know about photosynthesis?" |
| **Explain** | Understand a mechanism | "How does chlorophyll capture light?" |
| **Create** | Produce an artifact | "Write a diagram of the photosynthesis process" |
| **Resolve** | Answer a specific question | "What's the chemical equation for photosynthesis?" |

### How Goals Are Generated

1. **Read knowledge state** — What does the system currently "know"?
2. **Identify gaps** — What's missing, unclear, or unexplored?
3. **Generate candidates** — Propose 3-5 possible goals
4. **Score by priority** — Which goal is most important now?
5. **Select and commit** — Pursue top goal until done or stuck

## The Planner — How Goals Become Actions

Once a goal is selected, the planner figures out **how** to achieve it.

### Architecture

```python
class GoalPlanner(nn.Module):
    def __init__(self, config):
        # Reads goal + memory of past attempts
        self.encoder = TransformerEncoder(...)
        
        # Action selection
        self.action_head = nn.Linear(hidden_dim, num_actions)
        
        # Steering selection (which behavior to apply)
        self.steering_head = nn.Linear(hidden_dim, num_steering_vectors)
        
        # Continuation decision (keep going or revise?)
        self.continue_head = nn.Linear(hidden_dim, 2)
        
    def forward(self, goal, memory):
        # Encode state
        encoded = self.encoder(torch.cat([goal, memory], dim=-1))
        
        # Select actions
        action_probs = F.softmax(self.action_head(encoded), dim=-1)
        steering_probs = F.softmax(self.steering_head(encoded), dim=-1)
        continue_prob = torch.sigmoid(self.continue_head(encoded))
        
        return action_probs, steering_probs, continue_prob
```

### Action Types

| Action | Purpose | Example |
|--------|---------|---------|
| **Query** | Ask a specific question | "What is the role of ATP in photosynthesis?" |
| **Steer** | Apply behavior vector | Apply "technical" steering for precise output |
| **Generate** | Produce output | Generate explanation, diagram, summary |
| **Reflect** | Evaluate progress | "Am I closer to understanding this?" |

### Planning Loop

```python
def pursue_goal(agent, goal, max_steps=20):
    memory = agent.memory.get_goal_memory(goal)
    
    for step in range(max_steps):
        # Plan next action
        action, steering, should_continue = agent.planner(goal, memory)
        
        # Execute
        if action == 'query':
            output = agent.query(steering=steering)
        elif action == 'generate':
            output = agent.generate(steering=steering)
        elif action == 'reflect':
            progress = agent.evaluate_progress(goal, memory)
            if progress < threshold:
                return 'revise_goal'
        
        # Update memory
        memory.add(step, action, output, steering)
        
        # Check if goal achieved
        if agent.goal_achieved(goal, memory):
            return 'success'
        
        if not should_continue:
            return 'abandon'
    
    return 'timeout'
```

## Memory — Structured for Goal Pursuit

Not just a vector buffer. Memory needs to support **goal-directed behavior**.

### Three-Tier Structure

```python
class GoalMemory:
    def __init__(self, config):
        # Tier 1: Knowledge graph (facts discovered)
        self.knowledge_graph = KnowledgeGraph()
        
        # Tier 2: Attempt log (what was tried)
        self.attempt_log = deque(maxlen=100)
        
        # Tier 3: Goal history (pursued, achieved, abandoned)
        self.goal_history = deque(maxlen=50)
        
    def add_knowledge(self, subject, predicate, obj, confidence):
        """Add a fact to the knowledge graph."""
        self.knowledge_graph.add_edge(
            subject, predicate, obj,
            metadata={'confidence': confidence, 'source': 'discovery'}
        )
        
    def log_attempt(self, goal, action, output, success):
        """Log what was tried and whether it worked."""
        self.attempt_log.append({
            'goal': goal,
            'action': action,
            'output': output,
            'success': success,
            'timestamp': time.time()
        })
        
    def get_goal_memory(self, goal):
        """Retrieve memory relevant to current goal."""
        # Get related facts from knowledge graph
        related_facts = self.knowledge_graph.query(goal)
        
        # Get past attempts at similar goals
        similar_attempts = [
            a for a in self.attempt_log
            if self._goals_similar(a['goal'], goal)
        ]
        
        return self._compile_memory(related_facts, similar_attempts)
```

### What Memory Enables

| Memory Type | Enables |
|-------------|---------|
| **Knowledge graph** | Knows what it knows; identifies gaps |
| **Attempt log** | Doesn't repeat failed approaches |
| **Goal history** | Learns which goal types are achievable |

## Reward — Goal Achievement, Not Just Curiosity

The reward function must reflect **goal-driven** behavior, not just "interesting output."

### Components

```python
class GoalReward:
    def __init__(self, config):
        self.weights = {
            'goal_progress': 1.0,    # Did we get closer to the goal?
            'knowledge_gain': 0.5,   # Did we learn something new?
            'coherence': 0.3,        # Does it make sense?
            'artifact_quality': 0.5, # Is the output useful?
            'efficiency': 0.2,       # Did we achieve it efficiently?
        }
        
    def compute(self, goal, memory, output):
        # Goal progress (task-specific)
        progress = self._measure_progress(goal, memory)
        
        # Knowledge gain (new facts added to graph)
        knowledge_gain = self._measure_knowledge_gain(memory)
        
        # Coherence (perplexity, consistency)
        coherence = self._measure_coherence(output)
        
        # Artifact quality (task-specific evaluator)
        artifact_quality = self._evaluate_artifact(output, goal)
        
        # Efficiency (steps taken vs. typical)
        efficiency = self._measure_efficiency(goal, memory)
        
        total = (
            self.weights['goal_progress'] * progress +
            self.weights['knowledge_gain'] * knowledge_gain +
            self.weights['coherence'] * coherence +
            self.weights['artifact_quality'] * artifact_quality +
            self.weights['efficiency'] * efficiency
        )
        
        return total, {
            'progress': progress,
            'knowledge': knowledge_gain,
            'coherence': coherence,
            'quality': artifact_quality,
            'efficiency': efficiency
        }
```

### Goal Progress Measurement

Different goal types have different progress metrics:

| Goal Type | Progress Metric |
|-----------|-----------------|
| **Explore** | Coverage of topic space (facts discovered / estimated total) |
| **Explain** | Completeness of explanation (all sub-mechanisms covered) |
| **Create** | Artifact meets specification (evaluator score) |
| **Resolve** | Question answered with confidence (answer + confidence > threshold) |

### Phase 3: Evaluate Autonomy

```python
def evaluate_autonomy(agent, domain, num_goals=20):
    metrics = {
        'goals_completed': 0,
        'avg_steps_per_goal': 0,
        'knowledge_graph_size': 0,
        'artifact_quality': 0,
        'human_rating': 0,
    }
    
    for _ in range(num_goals):
        goal = goal_generator.generate(domain)
        result = pursue_goal(agent, goal)
        
        if result == 'success':
            metrics['goals_completed'] += 1
        metrics['avg_steps_per_goal'] += len(agent.memory.attempt_log)
        metrics['knowledge_graph_size'] += len(agent.memory.knowledge_graph)
        metrics['artifact_quality'] += artifact_evaluator(result.output)
        metrics['human_rating'] += human_evaluator(result.output)
    
    return {k: v / num_goals for k, v in metrics.items()}
```

**Success criteria:**
- Goals completed > 70%
- Human rating > 4/5
- Knowledge graph grows over time

## Honest Limitations

### What ALMA v5 Will NOT Do

| Claim | Reality |
|-------|---------|
| "Outperform fine-tuned models on benchmarks" | No — not designed for that |
| "Work without any training" | No — planner needs task-specific training |
| "Be fully general" | No — goal generator is domain-specific |
| "Replace human creativity" | No — augments, doesn't replace |

### When NOT to Use ALMA v5

- You need benchmark performance → **Fine-tune**
- You need single-turn responses → **Standard LLM**
- You can't tolerate multi-step latency → **Don't use**
- You don't need autonomy → **Prompt engineering is fine**

### When TO Use ALMA v5

- You need multi-step investigation
- You want self-directed exploration
- You need artifact creation with iteration
- You want the system to "think for itself"
