# ALMA v2 — Adaptive Language Model Agent

**From Frozen Separation to Fluid Collaboration**

A practical, demonstrable autonomous agent architecture that extracts maximum value from the ALMA research program while removing constraints that limit real-world utility.

---

## Executive Summary

**What we're keeping:**
- The autonomy loop (plan → act → observe → learn)
- Intrinsic motivation via multi-component reward
- Steering via activation addition (now optional)
- Memory-based novelty computation
- Mode switching for behavioral flexibility
- The DAW-inspired visualization philosophy

**What we're changing:**
- ❌ **Frozen belief engine** → ✅ **Adaptive collaboration** (fine-tune when it helps)
- ❌ **Single large LLM** → ✅ **Model cascades** (small fast + large slow)
- ❌ **Purely intrinsic reward** → ✅ **Hybrid reward** (intrinsic + task + human signals)
- ❌ **Fixed architecture** → ✅ **Composable modules** (swap planners, memories, rewards)
- ❌ **Research-only focus** → ✅ **Application-first design** (ship something useful)

**What we're adding:**
- LoRA adapters for efficient fine-tuning
- Distillation for model cascades
- Multi-LLM routing (local + API fallback)
- Structured memory (graph + vector + episodic)
- Curriculum learning with automatic difficulty adjustment
- Real-time human-in-the-loop intervention
- Exportable "skill packets" (trained planners + bridges)

---

## Core Insight: The Separation Was Never the Point

The frozen belief / trainable goal separation was a *methodological tool* for studying autonomous cognition, not an end in itself. The real value is:

> **Making language model behavior steerable, inspectable, and improvable through a lightweight intermediary that learns from experience.**

This reframing unlocks everything:
- Fine-tune the LLM when the planner can't solve the problem alone
- Use multiple LLMs of different sizes, routed by the planner
- Let humans provide reward signals alongside intrinsic curiosity
- Export trained planners as reusable "skills" for specific tasks

## Module Specifications

### 1. **Planner v2 — The Conductor**

**Purpose:** Make high-level decisions about *how* to solve a problem, not just *what* to steer toward.

**Architecture:**
```python
class PlannerV2(nn.Module):
    def __init__(self, config):
        # Core transformer (unchanged)
        self.transformer = TransformerEncoder(...)
        
        # NEW: Model selection head
        self.model_select = nn.Linear(hidden_dim, num_models)
        
        # NEW: LoRA trigger (binary: fine-tune or not)
        self.lora_trigger = nn.Linear(hidden_dim, 2)
        
        # NEW: Memory write gate (what to store)
        self.write_gate = nn.Linear(hidden_dim, memory_types)
        
        # Existing heads (enhanced)
        self.latent_goal = nn.Linear(hidden_dim, latent_dim)
        self.mode = nn.Linear(hidden_dim, num_modes)  # flat, hierarchical, delegate, refine
        self.alpha = nn.Linear(hidden_dim, 1)  # Now learned, not frozen
        self.temperature = nn.Linear(hidden_dim, 1)  # Control generation randomness
```

**Key innovations:**
- **Model selection:** Learn when to use the 135M model (fast) vs. 1.7B model (capable) vs. API call (smart)
- **LoRA triggering:** Learn when steering isn't enough and actual fine-tuning is needed
- **Memory gating:** Decide what's worth remembering (vector vs. graph vs. episodic)
- **PPO instead of REINFORCE:** Lower variance, faster convergence

**Training:**
- Policy gradients (PPO) for discrete choices (model, mode)
- Gradient descent for continuous outputs (alpha, temperature, latent goal)
- Auxiliary losses for reconstruction (predict outcome from goal)

### 2. **Bridge v2 — The Translator**

**Purpose:** Translate between planner intentions and model activations — now with optional fine-tuning.

**Architecture:**
```python
class BridgeV2(nn.Module):
    def __init__(self, config):
        # Existing projections
        self.up_projection = nn.Linear(latent_dim, model_dim)
        self.down_projection = nn.Linear(model_dim, latent_dim)
        
        # NEW: Multi-layer steering (per-layer alpha)
        self.layer_alphas = nn.Parameter(torch.ones(num_layers))
        
        # NEW: LoRA adapter generator
        self.lora_generator = nn.Sequential(
            nn.Linear(latent_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, rank * 2)  # A and B matrices
        )
        
        # NEW: Steering vector library (learned prototypes)
        self.steering_prototypes = nn.Parameter(torch.randn(num_prototypes, model_dim))
```

**Key innovations:**
- **Multi-layer steering:** Inject at multiple depths with learned magnitudes
- **LoRA generation:** Produce low-rank adapters on-the-fly from latent goals
- **Prototype steering:** Reuse learned steering vectors for common tasks (style, formality, domain)

**Fine-tuning integration:**
```python
def apply_steering(self, model, latent_goal, use_lora=False):
    if use_lora:
        # Generate LoRA weights and apply
        lora_weights = self.lora_generator(latent_goal)
        apply_lora(model, lora_weights, rank=8)
    else:
        # Standard activation addition
        steering = self.get_steering_vector(latent_goal)
        inject_at_layers(model, steering, self.layer_alphas)
```

### 3. **Model Router — The Traffic Controller**

**Purpose:** Dynamically select which model to use based on task complexity, latency requirements, and cost.

**Architecture:**
```python
class ModelRouter(nn.Module):
    def __init__(self, planners, models):
        self.models = nn.ModuleList(models)  # [SmolLM2-135M, SmolLM2-1.7B, APIWrapper]
        self.cost_table = [0.0, 0.0, 0.002]  # $ per token (API costs money)
        self.latency_table = [10, 50, 200]   # ms per token (estimated)
        
    def select(self, task_embedding, budget, latency_constraint):
        # Planner outputs distribution over models
        model_logits = self.planner.model_select(task_embedding)
        
        # Mask out models that violate constraints
        mask = (self.cost_table <= budget) & (self.latency_table <= latency_constraint)
        model_logits[~mask] = -float('inf')
        
        return Categorical(logits=model_logits).sample()
```

**Model cascade strategy:**
1. Try 135M model first (fast, free)
2. If coherence < threshold or task complexity > threshold, escalate to 1.7B
3. If still failing or task requires external knowledge, use API
4. Log all escalations; train planner to predict when escalation is needed

**Distillation for efficiency:**
- Run complex tasks on 1.7B
- Distill outputs into 135M training data
- Periodically fine-tune 135M on distilled data
- Planner learns to use 135M for more tasks over time

### 4. **Memory Fabric — The Knowledge Layer**

**Purpose:** Store and retrieve different types of information efficiently.

**Three-tier architecture:**

| Type | Structure | Use Case | Query Speed |
|------|-----------|----------|-------------|
| **Vector** | FAISS/HNSW index | Semantic similarity, novelty computation | ~1ms |
| **Graph** | NetworkX + embeddings | Causal relationships, concept maps | ~10ms |
| **Episodic** | Temporal buffer + summaries | Full trajectories, replay learning | ~100ms |

**Vector Memory (fast novelty):**
```python
class VectorMemory:
    def __init__(self, dim=768, size=10000):
        self.index = faiss.IndexHNSWFlat(dim, 32)
        self.storage = []  # Keep actual vectors for retrieval
        
    def add(self, embedding, metadata):
        self.index.add(embedding)
        self.storage.append((embedding, metadata))
        
    def novelty(self, embedding):
        D, I = self.index.search(embedding, k=5)
        return 1.0 - D.mean()  # Distance = novelty
```

**Graph Memory (structured knowledge):**
```python
class GraphMemory:
    def __init__(self):
        self.graph = nx.DiGraph()
        self.node_encoder = SentenceTransformer('all-MiniLM-L6-v2')
        
    def add_relation(self, subject, predicate, object):
        # Create or update nodes
        for node in [subject, object]:
            if node not in self.graph:
                self.graph.add_node(node, embedding=self.node_encoder.encode(node))
        self.graph.add_edge(subject, object, relation=predicate)
        
    def query(self, question):
        # Embed question, find relevant subgraph
        q_emb = self.node_encoder.encode(question)
        # ... graph traversal with semantic scoring
```

**Episodic Memory (full trajectories):**
```python
class EpisodicMemory:
    def __init__(self, max_episodes=100):
        self.episodes = deque(maxlen=max_episodes)
        
    def store(self, trajectory):
        # trajectory = [(state, action, reward, next_state), ...]
        self.episodes.append({
            'trajectory': trajectory,
            'total_reward': sum(r for _,_,r,_ in trajectory),
            'summary': self.summarize(trajectory)
        })
        
    def sample_for_replay(self, batch_size=32):
        # Prioritized sampling by reward
        priorities = [e['total_reward'] for e in self.episodes]
        return random.choices(self.episodes, weights=priorities, k=batch_size)
```

**Memory read/write policy:**
- Planner outputs `write_gate` vector: which memories to update
- Vector: always write outcomes (for novelty)
- Graph: write when causal structure detected (high coherence + multi-step)
- Episodic: write when trajectory is unusual (high reward or high novelty)

### 5. **Reward Console — The Motivation Mixer**

**Purpose:** Combine multiple reward signals into a coherent learning signal.

**Six reward channels:**

| Channel | Source | Weight | Purpose |
|---------|--------|--------|---------|
| **Intrinsic** | Computed internally | 1.0 | Curiosity, novelty, coherence |
| **Task** | Goal achievement | 0.5–2.0 | Directed exploration, completion |
| **Human** | RLHF-style feedback | 0.1–5.0 | Alignment, quality, safety |
| **Social** | Multi-agent consensus | 0.3–1.0 | Agreement, debate resolution |
| **Curriculum** | Automatic difficulty | 0.2–0.5 | Progressive learning |
| **Safety** | Guardrail violations | -10.0 | Hard constraints |

**Implementation:**
```python
class RewardConsole(nn.Module):
    def __init__(self, config):
        # Learnable weights (can be fixed or trained)
        self.weights = nn.ParameterDict({
            'intrinsic': nn.Parameter(torch.tensor(1.0)),
            'task': nn.Parameter(torch.tensor(1.0)),
            'human': nn.Parameter(torch.tensor(2.0)),
            'social': nn.Parameter(torch.tensor(0.5)),
            'curriculum': nn.Parameter(torch.tensor(0.3)),
            'safety': nn.Parameter(torch.tensor(-10.0))
        })
        
        # Individual reward computers
        self.intrinsic = IntrinsicReward(config)
        self.task = TaskReward(config)
        self.human = HumanFeedbackBuffer()
        # ...
        
    def compute(self, state, action, outcome, context):
        rewards = {
            'intrinsic': self.intrinsic(state, outcome),
            'task': self.task(outcome, context.goal),
            'human': self.human.get_reward(outcome),
            'social': self.social.get_consensus_reward(outcome),
            'curriculum': self.curriculum.get_difficulty_adjusted_reward(outcome),
            'safety': self.safety.check_violations(outcome)
        }
        
        # Weighted sum with optional normalization
        total = sum(self.weights[k] * r for k, r in rewards.items())
        return total, rewards  # Return decomposition for debugging
```

**Human-in-the-loop:**
- Click/touch interface to provide +/− feedback on outputs
- Slider for continuous reward (like/dislike intensity)
- Natural language feedback → embedded → reward bonus
- Active learning: request human feedback when uncertain (high entropy)

**Curriculum learning:**
- Track success rate on tasks
- Automatically adjust difficulty (goal complexity, coherence threshold)
- Reward = base_reward × difficulty_multiplier
- Difficulty multiplier > 1 for hard tasks, < 1 for easy tasks

## Applications & Demonstrations

### 1. **Concept Explorer** (Knowledge Mapping)

**Goal:** Map what an LLM "knows" about a topic by steering toward it and recording trajectories.

**Setup:**
- Model: SmolLM2-360M (local, fast)
- Reward: 0.5× novelty + 0.5× relevance to target concept
- Memory: Vector + Graph (build concept map as it explores)

**Demo flow:**
1. User enters concept: "photosynthesis"
2. Agent explores for 100 steps, building a concept graph
3. Visualization shows: concept nodes, edges, confidence scores
4. Export: Interactive knowledge map (HTML/D3)

**Why it works:** Combines steering (activation addition) with structured memory (graph) to produce something more useful than raw text generation.

---

### 2. **Style Shifter** (Register Control)

**Goal:** Generate text in different styles without fine-tuning the base model.

**Setup:**
- Model: SmolLM2-135M (fast iteration)
- Reward: Style classifier score + coherence
- Bridge: Pre-trained style steering vectors (formal, casual, technical, narrative)

**Demo flow:**
1. User enters topic: "climate change"
2. User selects style: "technical" → "casual" → "narrative"
3. Agent generates same content in each style
4. Visualization shows: steering vector magnitudes, style classifier confidence

**Why it works:** Steering vectors are perfect for style transfer — they shift register without changing core knowledge.

---

### 3. **Task Delegator** (Model Cascade)

**Goal:** Automatically route tasks to the right model (fast vs. capable vs. expensive).

**Setup:**
- Models: [SmolLM2-135M, SmolLM2-1.7B, Claude API]
- Planner: Learns to predict which model will succeed
- Reward: Task success − cost − latency

**Demo flow:**
1. User enters tasks of varying complexity
2. Agent routes each task, logs model choice and outcome
3. After 50 tasks, show: routing accuracy, cost savings, latency distribution
4. Compare to baseline (always use largest model)

**Why it works:** Demonstrates practical value — same performance at lower cost/latency.

---

### 4. **Skill Pack Creator** (Exportable Planners)

**Goal:** Train a planner for a specific task, export it as a reusable "skill."

**Setup:**
- Task: "Generate coherent multi-paragraph explanations"
- Training: 1000 steps with depth bonus + coherence reward
- Export: Planner weights + bridge + steering library

**Demo flow:**
1. Train planner on explanation generation
2. Export as `skill_explanation_v1.pt`
3. Load skill in new session, apply to different topics
4. Show: transfer performance (no retraining needed)

**Why it works:** Turns the research into a product — reusable, composable intelligence modules.

---

### 5. **Debate Moderator** (Multi-Agent Social Reward)

**Goal:** Run two agents with different reward functions, let them debate, extract consensus.

**Setup:**
- Agent A: Reward = novelty + technical accuracy
- Agent B: Reward = novelty + accessibility
- Social reward: Agreement bonus, contradiction penalty

**Demo flow:**
1. Enter topic: "nuclear energy"
2. Agents debate for 50 rounds
3. Visualization shows: convergence over time, remaining disagreements
4. Output: Balanced summary with pros/cons

**Why it works:** Demonstrates multi-agent dynamics and social reward — more interesting than single-agent exploration.

### Phase 2: PPO Upgrade (Week 3)

**Goal:** Replace REINFORCE with PPO for stable training.

**Changes:**
- Add value head to planner: `self.value_head = nn.Linear(hidden_dim, 1)`
- Implement GAE (Generalized Advantage Estimation)
- Add clipped policy ratio: `L^CLIP = E[min(r_t * A_t, clip(r_t, 1-ε, 1+ε) * A_t)]`
- Value loss: `L^VF = E[(V(s_t) - V_target)^2]`

**Expected improvement:** 2–3× faster convergence, less oscillation.

## Technical Requirements

**Minimum (commodity GPU):**
- GPU: RTX 3060 (12GB) or equivalent
- RAM: 16GB
- Storage: 50GB (models + memory indices)

**Recommended:**
- GPU: RTX 4090 (24GB) or A100 (40GB)
- RAM: 32GB
- Storage: 200GB SSD

**Model sizes:**
- SmolLM2-135M: ~300MB (quantized), ~600MB (full)
- SmolLM2-360M: ~800MB (quantized), ~1.5GB (full)
- SmolLM2-1.7B: ~3.5GB (quantized), ~7GB (full)

**Memory footprint:**
- Vector memory (10k items, 768-dim): ~30MB
- Graph memory (1k nodes, 10k edges): ~50MB
- Episodic memory (100 episodes, 100 steps each): ~200MB

**Training speed (RTX 3060):**
- PPO step (batch=8, 8 tokens): ~50ms
- Full loop (plan + steer + generate + learn): ~200ms
- Steps per hour: ~18,000
